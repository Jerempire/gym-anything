# Reading Readiness Assessment

## Task Overview

**Occupation**: Literacy Specialist / Kindergarten Teacher
**Difficulty**: Hard
**Timeout**: 600 seconds / 70 max steps

## Domain Context

Literacy specialists in early childhood education regularly evaluate software tools to determine their suitability for kindergarten reading readiness programs. This involves examining every available language activity, personally trying each one to assess its instructional quality, and producing a structured report that guides purchasing and curriculum decisions.

## Goal

Produce a **Reading Readiness Assessment Report** at `~/Desktop/reading_readiness_report.txt` after:
1. Exploring all three Language sub-tabs: **Letters**, **Words**, **Vocabulary**
2. Interacting with at least 4 different language activities
3. Documenting each activity with its name, skill focus, and age-appropriateness rating

## Success Criteria

The report file `~/Desktop/reading_readiness_report.txt` must:
- Be created after the task started
- Be at least 400 bytes
- Contain labeled sections for **Letters**, **Words**, and **Vocabulary**
- Mention specific GCompris language activity names

## Scoring (100 points)

| Criterion | Points |
|-----------|--------|
| Report file exists | 10 |
| Report created after task started (gate) | 15 |
| Report is ≥400 bytes | 10 |
| Letters section present | 20 |
| Words section present | 20 |
| Vocabulary section present | 10 |
| 3+ language activity keywords | 15 |

Pass threshold: **60 points**

## GCompris Language Activities (Reference)

**Letters tab**: Baby keyboard, A baby word processor, Draw letters, Alphabet sequence, Click on a lowercase letter, Click on an uppercase letter, Simple letters
**Words tab**: Activities about word recognition and building
**Vocabulary tab**: Activities about vocabulary development

## Notes

- The Language section is accessed via the cow/ABC icon in GCompris's top category bar
- Each sub-tab has its own content grid
