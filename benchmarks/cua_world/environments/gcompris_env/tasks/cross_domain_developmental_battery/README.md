# Cross-Domain Developmental Battery

## Task Overview

**Occupation**: Child Development Psychologist
**Difficulty**: Very Hard
**Timeout**: 720 seconds / 80 max steps

## Domain Context

Child development psychologists construct standardized assessment batteries to evaluate children across multiple cognitive domains simultaneously. Using a software tool like GCompris for this purpose requires systematically covering all major developmental areas: numerical reasoning, language/literacy, scientific thinking, and spatial/logical problem-solving. The psychologist must personally experience each activity to understand what it measures and whether it is valid for assessment purposes.

## Goal

Produce a **Multi-Domain Developmental Assessment Battery** report at `~/Desktop/developmental_assessment_battery.txt` after:
1. Completing one activity from the Math/Numbers category (mathematical reasoning)
2. Completing one activity from the Language/ABC category (language/literacy)
3. Completing one activity from the Science/Experiment sub-tab (scientific inquiry)
4. Completing one activity from the Games category (spatial/logical thinking)

## Success Criteria

The report file `~/Desktop/developmental_assessment_battery.txt` must:
- Be created after the task started
- Be at least 500 bytes
- Contain content covering all four domains: math, language, science, and games
- Mention specific activity names from at least three of the four domains

## Scoring (100 points)

| Criterion | Points |
|-----------|--------|
| Report file exists | 10 |
| Report created after task started (gate) | 10 |
| Report is ≥500 bytes | 10 |
| Math domain documented (arithmetic/number keywords) | 20 |
| Language domain documented (letter/word/alphabet keywords) | 20 |
| Science domain documented (experiment/color/gravity keywords) | 20 |
| Games domain documented (maze/memory/game keywords) | 10 |

Pass threshold: **60 points**

## Notes

- The task requires navigating to FOUR different GCompris categories
- The agent must actually complete activities in each domain
- Report must contain domain-specific content demonstrating genuine engagement
- Very hard due to multi-category navigation and breadth of coverage required
