FreshCart Online Grocery — Analytics Remediation Report
=======================================================

Prepared by: DataFlow Consulting, LLC
Site:        FreshCart Online Grocery
Date:        2026-03-15
Priority:    Critical — resolve all issues before weekly reporting resumes

Background
----------
FreshCart Online Grocery hired a contractor to set up Matomo analytics
for their e-commerce website. The contractor left before completing the
work. An independent audit has identified five issues that must be
corrected before the marketing team can rely on the data.


ISSUE 1: Incorrect Conversion Goal Patterns
--------------------------------------------
Two of the four e-commerce funnel goals have wrong configurations.
The other two are correct and must NOT be modified.

  Goal "Add to Cart":
    Current state : URL pattern is "/cart" (match type: contains)
    Problem       : Too broad — matches /cart, /cart/view, /cart/empty, etc.
    Required fix  : Change URL pattern to "/cart/add" (keep match type as contains)

  Goal "Begin Checkout":
    Current state : URL pattern is "/checkout" (match type: exact match)
    Problem       : Exact match fails for /checkout/step1, /checkout/payment, etc.
    Required fix  : Change match type from "exact match" to "contains"
                    (keep URL pattern as "/checkout")

  Do NOT modify goals "Product Page View" or "Purchase Complete" — they
  are configured correctly.


ISSUE 2: Broken Visitor Segment
-------------------------------
  Segment "Returning Customers":
    Current state : Condition is "Number of Visits greater than 5"
    Problem       : This filters for frequent visitors, not returning visitors.
                    A visitor who came back once would be missed entirely.
    Required fix  : Change the condition to "Visitor Type equals Returning"


ISSUE 3: Missing Visitor Segment
---------------------------------
  The segment "Mobile Shoppers" was planned but never created.

    Name       : Mobile Shoppers
    Condition  : Device type equals Smartphone
    Visibility : Visible to all users

  Create this segment.


ISSUE 4: Dashboard Widget Errors
---------------------------------
  The "Weekly Performance" dashboard exists but contains two incorrect
  widgets. The marketing team needs goal and device data, not page URL
  lists or screen resolution breakdowns.

    Replace "Page URLs" widget      -->  "Goals Overview"
           (currently: Actions module)    (correct: Goals module)

    Replace "Screen Resolution" widget  -->  "Device Type"
           (currently: Resolution module)     (correct: Devices Detection module)

  The "Visits Over Time" and "Referrer Types" widgets are correct.
  Do NOT remove or modify them.


ISSUE 5: Missing Custom Dimension
----------------------------------
  A visit-scope custom dimension named "Customer Tier" was planned to
  segment analytics by subscription level (Free, Premium, Enterprise).
  It was never created.

    Name   : Customer Tier
    Scope  : Visit
    Status : Active

  Create and activate this custom dimension.


End of Report
