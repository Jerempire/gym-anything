# Task: FDA SaMD Regulatory Research

## Overview

**Difficulty**: Very Hard
**Occupation**: Regulatory Affairs Manager
**Domain**: Medical Device Regulatory Affairs

## Background

Regulatory Affairs Managers at medical device companies spend significant time researching FDA requirements for novel product categories. Software as a Medical Device (SaMD) is one of the fastest-growing regulatory areas, with evolving guidance from FDA, international bodies (IMDRF), and other regulators. A RA manager preparing a pre-submission package must synthesize requirements from multiple authoritative sources.

## Task Goal

Research the regulatory landscape for AI/ML-based Software as a Medical Device (SaMD) from three authoritative sources: FDA, the Federal Register, and IMDRF. Organize findings in Firefox bookmarks and produce a regulatory requirements matrix.

## Key Sources

### FDA Digital Health Center of Excellence
- FDA guidance on Software as a Medical Device
- AI/ML-Based Software as a Medical Device Action Plan (2021)
- Predetermined Change Control Plans (PCCPs)
- URL: https://www.fda.gov/medical-devices/digital-health-center-excellence

### Federal Register
- Published rules, proposed rules, and notices related to medical device software
- URL: https://www.federalregister.gov (search: "software medical device" or "SaMD")

### IMDRF (International Medical Device Regulators Forum)
- SaMD: Key Definitions (N10)
- SaMD: Clinical Evaluation (N41)
- URL: https://www.imdrf.org/documents

## Success Criteria

### History Evidence
- FDA.gov visited ≥3 times (different pages for guidance depth)
- FederalRegister.gov visited ≥1 time
- IMDRF.org visited ≥1 time

### Bookmark Collection
- "FDA Regulatory Research" bookmark folder with ≥6 bookmarks
- Bookmarks should span all 3 sources (FDA, Federal Register, IMDRF)

### Downloaded Files
- At least 1 PDF downloaded from a .gov or regulatory domain
- File must be >10KB (not an empty/failed download)

### Regulatory Matrix File
- File: `~/Documents/samd_regulatory_matrix.txt`
- Created after task start
- Contains "SaMD", "FDA", and "IMDRF" keywords
- Contains regulatory/medical device terminology

## Verification Strategy

1. Firefox history for all 3 required domains (25 pts)
2. "FDA Regulatory Research" bookmark folder with ≥6 bookmarks (20 pts)
3. PDF download from regulatory domain >10KB (20 pts)
4. Matrix file exists and is recent (15 pts)
5. Matrix file content quality: required keywords + regulatory terms (20 pts)

**Pass threshold**: 60/100 points
