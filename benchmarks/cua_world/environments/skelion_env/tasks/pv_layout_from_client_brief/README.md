# Task: pv_layout_from_client_brief

## Occupation Context

**Primary occupation**: Solar Energy Systems Engineer
**GDP relevance**: $2.28B (Skelion, solar design and simulation category)
**Workflow**: Translating a client brief into a complete rooftop PV layout, with all system parameters configured correctly and a deliverable project report exported.

---

## Task Overview

A commercial property owner in San Francisco, CA has contracted SunPath Solutions for a rooftop PV system design. The engineer (agent) receives a client brief specifying exact system parameters and must deliver a configured SketchUp model with panels placed, and a CSV project report exported via Skelion.

The building model `Solar_Project.skp` is open in SketchUp Make 2017 with the Skelion plugin active and toolbar visible.

---

## Goal (End State)

The following conditions must all be satisfied:

1. **Geographic location** set to San Francisco, CA — latitude 37.7314° N, longitude 122.3850° W (within ±0.20° tolerance)
2. **Panel orientation**: Landscape
3. **Panel tilt**: 5 degrees
4. **Row spacing**: 1.5 meters
5. **Minimum 75 solar panels** placed on the building roof
6. **`PV_Layout_Report.csv`** exported to `C:\Users\Docker\Desktop\` — non-empty CSV report generated by Skelion

---

## Client Brief Location

`C:\Users\Docker\Desktop\client_brief.txt` — placed by setup_task.ps1

This document contains all required parameters. The agent must read and implement all of them.

---

## Difficulty Rationale (very_hard)

- Agent receives only a goal + client brief; no UI path is provided
- Must configure 4 separate Skelion parameters (orientation, tilt, row spacing) AND set geographic location through SketchUp's Model Info dialog
- Must discover Skelion's CSV export function independently (not named in description)
- CSV export is a multi-step workflow requiring the right Skelion toolbar button
- 75+ panels requires correct use of the placement tool, not just a single click

## Setup Details

`setup_task.ps1` performs:
1. Clears leftover output files from previous runs
2. Launches SketchUp with a one-shot Ruby plugin to record baseline ComponentInstance count → saves to `C:\Users\Docker\baseline_comp_count.json`
3. Kills SketchUp, removes plugin
4. Writes `client_brief.txt` to Desktop with all required parameters
5. Calls `Reset-SketchUpModel` to launch SketchUp for the agent

---

## Export Details

`export_result.ps1` performs:
1. Saves model via Ctrl+S
2. Checks Desktop for `PV_Layout_Report.csv`
3. Kills SketchUp
4. Injects Ruby export plugin, relaunches SketchUp (plugin fires at 10s, writes `model_state_export.json`)
5. Kills SketchUp, removes plugin
6. Reads baseline + export JSON, computes panel delta
7. Writes `C:\Users\Docker\pv_layout_result.json`

## Edge Cases

- If agent uses File > Save As instead of Ctrl+S, the `export_result.ps1` still relaunches with `Solar_Project.skp` to extract location/count — this is fine since Save As produces a copy
- If Skelion places panels inside Groups, the recursive ComponentInstance counter still counts them
- Partial credit for panels: if agent places 40/75, they receive proportional points up to 20 pts max
