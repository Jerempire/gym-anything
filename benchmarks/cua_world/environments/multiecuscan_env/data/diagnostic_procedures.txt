========================================================================
STANDARD AUTOMOTIVE DIAGNOSTIC PROCEDURES
For use with Multiecuscan OBD-II Diagnostic Software
Based on SAE J1979 / ISO 15031-5 Standards
========================================================================

1. INITIAL VEHICLE IDENTIFICATION
------------------------------------------------------------------------
Before performing diagnostics, record:
  - Vehicle Make, Model, Year
  - VIN (Vehicle Identification Number)
  - Engine code (stamped on engine block or on vehicle data plate)
  - Mileage / odometer reading
  - Customer complaint description

In Multiecuscan:
  a) Select the correct Make from Panel 1 (left)
  b) Select the Model and Version from Panel 2
  c) Select the System Type from Panel 3 (Engine, ABS, Airbag, etc.)
  d) Select the specific ECU Module from Panel 4 (right)

2. ECU CONNECTION AND IDENTIFICATION
------------------------------------------------------------------------
Procedure:
  a) Click "Connect" (or "Simulate" for practice/demo mode)
  b) Wait for connection to establish (Info screen loads automatically)
  c) Record from Info screen:
     - ECU Part Number
     - ECU Hardware Version
     - ECU Software Version
     - ECU Calibration ID
     - System Status (active/inactive flags)

Key shortcut: Default screen after connection is Info

3. ERROR CODE (DTC) READING AND ANALYSIS
------------------------------------------------------------------------
Procedure:
  a) Press F3 or click "Errors" tab
  b) Wait for DTC scan to complete
  c) Record all stored DTCs (Diagnostic Trouble Codes)
  d) Note DTC status: Active (current) vs Stored (historical)
  e) Cross-reference each DTC with the OBD-II code database

DTC Code Format:
  - First character: P (Powertrain), B (Body), C (Chassis), U (Network)
  - Second character: 0 (SAE standard), 1 (Manufacturer-specific)
  - Characters 3-5: Specific fault identification

Common FCA-specific DTCs:
  P0100-P0104  Mass Air Flow sensor circuit
  P0110-P0114  Intake Air Temperature sensor
  P0115-P0119  Engine Coolant Temperature sensor
  P0130-P0167  O2 Sensor circuits
  P0170-P0175  Fuel trim out of range
  P0200-P0206  Injector circuits
  P0300-P0312  Misfire detected
  P0400-P0409  EGR system
  P0420-P0424  Catalytic converter efficiency
  P0480-P0485  Cooling fan circuits
  P2000-P2099  NOx and aftertreatment
  P2400-P2499  EVAP system

  U0073  Control Module Communication Bus Off
  U0100  Lost Communication with ECM/PCM
  U0121  Lost Communication with Anti-Lock Brake System
  U0140  Lost Communication with Body Control Module
  U1602  CAN communication fault (FCA-specific)

4. LIVE DATA PARAMETER MONITORING
------------------------------------------------------------------------
Procedure:
  a) Press F4 or click "Parameters" tab
  b) Parameter list displays all available PIDs for the connected ECU
  c) Select parameters using checkbox/spacebar (max 4 in free version)
  d) Values update in real-time in the table

Critical Engine Parameters to Monitor:
  - Engine RPM: Normal idle 650-900 rpm (varies by engine)
  - Coolant Temperature: Normal operating 80-100°C
  - Battery/System Voltage: Normal 13.5-14.5V (engine running)
  - Throttle Position: Idle 10-20%, WOT 90-100%
  - MAF Air Flow: Idle 2-8 g/s (varies by engine displacement)
  - Intake Air Temperature: Ambient + 10-20°C
  - Engine Load: Idle 15-30%, acceleration varies
  - Fuel Trim (STFT/LTFT): Normal ±5% (>±10% indicates problem)
  - O2 Sensor Voltage: Switching 0.1-0.9V (closed loop)
  - EGR Commanded: 0% at idle, varies with load
  - Fuel Rail Pressure: Diesel 250-1600 bar (varies with load)

5. GRAPHICAL DATA ANALYSIS
------------------------------------------------------------------------
Procedure:
  a) Select parameters in the Parameters screen (F4)
  b) Press F5 or click "Graph" tab
  c) Graph displays selected parameters over time
  d) Maximum 1 graph in free version, up to 4 in registered
  e) Use graph to identify:
     - Sudden parameter changes (potential fault)
     - Oscillation patterns (sensor problems)
     - Correlation between parameters
     - Drift over time (gradual degradation)

6. ACTUATOR TESTING (Registered Version Only)
------------------------------------------------------------------------
Procedure:
  a) Press F6 or click "Actuators" tab
  b) Available actuator tests depend on ECU module:
     - Injector balance test
     - EGR valve activation
     - Cooling fan relay test
     - Purge solenoid activation
     - DPF regeneration initiation
  c) Monitor related parameters during test
  d) Compare results against manufacturer specifications

7. CONFIGURATION AND ADJUSTMENTS
------------------------------------------------------------------------
Procedure:
  a) Press F7 or click "Adjustments" tab
  b) Available adjustments depend on ECU module:
     - Service interval reset
     - DPF regeneration counter reset
     - Throttle body adaptation
     - Key programming acknowledgment
  c) PROXI alignment: Press Ctrl+Alt+C to view PROXI bytes
  d) ALWAYS back up current configuration before making changes

Body Computer Configurable Items (FCA vehicles):
  - Daytime Running Lights (DRL) mode/brightness
  - Door lock behavior (auto-lock speed, unlock pattern)
  - Interior light timeout duration
  - Seatbelt warning buzzer duration
  - Follow-me-home headlight duration
  - Key fob window control (up/down)
  - Rain sensor sensitivity
  - One-touch indicator flash count

8. MULTI-SYSTEM DIAGNOSTIC SCAN
------------------------------------------------------------------------
Procedure for comprehensive vehicle health check:
  a) Scan Engine ECU → record DTCs
  b) Disconnect (F11), select Transmission → scan
  c) Disconnect, select ABS/ESP → scan
  d) Disconnect, select Airbag/SRS → scan
  e) Disconnect, select Body Computer → scan
  f) Disconnect, select Steering → scan (if applicable)
  g) Compile all findings into consolidated report

Report should include:
  - Vehicle identification
  - Date and time of diagnostic session
  - Per-system: ECU info, DTCs found, DTC descriptions
  - Parameter readings (key values from each system)
  - Recommended actions for each fault found
  - Overall vehicle health assessment

9. SESSION DATA EXPORT
------------------------------------------------------------------------
Multiecuscan data logging:
  - Registered version auto-saves session data
  - CSV export with configurable separator
  - Data includes timestamps and all monitored parameter values
  - Typical CSV format:
    Time;Parameter1 [unit];Parameter2 [unit];...
    0.000;value1;value2;...

10. DIAGNOSTIC REPORT FORMAT
------------------------------------------------------------------------
Standard diagnostic report structure:

VEHICLE DIAGNOSTIC REPORT
=========================
Date: [YYYY-MM-DD HH:MM]
Vehicle: [Make] [Model] [Year] [Engine Code]
Mileage: [km/miles]
Technician: [Name]

ECU IDENTIFICATION
------------------
Module: [ECU Name]
Part Number: [PN]
Hardware Version: [HW]
Software Version: [SW]

DIAGNOSTIC TROUBLE CODES
-------------------------
Code    | Status  | Description
--------|---------|------------------------------------------
P0420   | Active  | Catalyst System Efficiency Below Threshold
P0171   | Stored  | System Too Lean (Bank 1)

LIVE DATA SNAPSHOT
------------------
Parameter                  | Value    | Normal Range | Status
--------------------------|----------|--------------|-------
Engine RPM                | 750 rpm  | 650-900 rpm  | OK
Coolant Temperature       | 88°C     | 80-100°C     | OK
Battery Voltage           | 14.2V    | 13.5-14.5V   | OK
Throttle Position         | 15.3%    | 10-20%       | OK

RECOMMENDATIONS
---------------
1. P0420 - Inspect catalytic converter, check O2 sensor signals
2. P0171 - Check for vacuum leaks, inspect MAF sensor, check fuel pressure

OVERALL ASSESSMENT: [Pass/Fail/Attention Required]
